## Design

The packet should be practical and rehearsal-friendly. It should help Changqi explain the education data project honestly, defend the statistics and code at a working level, connect his CS and teaching background to undergraduate outcomes analytics, and answer likely technical and behavioral questions.

The materials will avoid overstating causal claims. They will frame the GitHub project as an AI-assisted portfolio project that Changqi now understands and can defend, while making clear that sensitive institutional data should not be placed into external AI tools.

## Files

- `README.md`: Start-here index and interview priorities.
- `01_two_day_plan.md`: Monday/Tuesday/Wednesday preparation schedule.
- `02_project_mastery.md`: Project walkthrough, technical concepts, limitations, and honest AI-assisted framing.
- `03_possible_questions.md`: Interview questions with answer templates.
- `04_vt_uaa_fit.md`: Role fit, VT context, and questions to ask.
- `05_mock_interview_script.md`: Timed 30-45 minute mock interview.
- `06_reply_email.md`: Short response email accepting the interview.

## Success Criteria

- Changqi can give a 60-90 second personal pitch.
- Changqi can explain each GitHub project script and the core statistical model.
- Changqi can answer questions about SQL, R/tidyverse, missing data, hierarchical models, and higher-ed outcomes analysis.
- Changqi has a respectful answer about using ChatGPT as a learning and pair-programming tool.
- Changqi has a concrete two-day practice plan and a ready-to-send email reply.
