# Stage 1: Screening Call Practice

Stage 1 is a 30-minute rapid-fire technical screening. You are **not** allowed to use AI during this call. You need to know these concepts well enough to explain them in 1-2 sentences.

## 🚀 Rapid-Fire Questions

### 1. The Web & HTTP
- **Q: What is the difference between GET and POST?**
  - *A:* GET is for requesting data from a server (it's visible in the URL). POST is for sending data to a server, typically to create or update something (it's hidden in the request body).
- **Q: What does a 404 error mean? A 500 error?**
  - *A:* 404 means "Not Found" (client error). 500 means "Internal Server Error" (something broke on the server).
- **Q: What is JSON?**
  - *A:* JavaScript Object Notation. It's a lightweight text format for storing and transporting data (the "language" APIs use to talk).

### 2. Frontend Basics
- **Q: What is the difference between HTML, CSS, and JavaScript?**
  - *A:* HTML is the skeleton (structure), CSS is the skin (styling), and JavaScript is the brain (interactivity).
- **Q: What is a React Component?**
  - *A:* A reusable piece of the UI (like a Button or a Navigation bar) that manages its own logic and appearance.
- **Q: What is "State" in React?**
  - *A:* An object that holds data that might change over time (e.g., search text, login status) and triggers a re-render when it changes.

### 3. Backend & Data
- **Q: What is an API?**
  - *A:* Application Programming Interface. A set of rules that allows two applications to talk to each other.
- **Q: Difference between SQL and NoSQL?**
  - *A:* SQL (e.g., PostgreSQL) uses tables and fixed schemas (structured). NoSQL (e.g., MongoDB) stores data as documents (flexible).
- **Q: What is a Database Migration?**
  - *A:* A way to manage changes to your database schema over time without losing data.

### 4. General Engineering
- **Q: What is Git?**
  - *A:* A version control system that tracks changes in your code and allows collaboration.
- **Q: Explain "Dry" principle.**
  - *A:* "Don't Repeat Yourself" – reducing repetition of code patterns by using functions and components.
